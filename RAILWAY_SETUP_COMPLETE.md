## 📝 Next Steps

### Step 1: Update OAuth Credentials
1. Go to Railway Dashboard: https://railway.com/project/fa12ff46-d3c3-4f1d-a49c-0957dc9fae5f
2. Select **hotelbook-api** service
3. Go to **Variables** tab
4. Update:
   - `GOOGLE_CLIENT_ID` - Get from: https://console.cloud.google.com
   - `GOOGLE_CLIENT_SECRET` - Get from: https://console.cloud.google.com
   - `GITHUB_CLIENT_ID` - Get from: GitHub Settings → Developer settings → OAuth Apps
   - `GITHUB_CLIENT_SECRET` - Get from: GitHub Settings → Developer settings → OAuth Apps

### Step 2: Update M-Pesa Credentials
1. In Railway Variables, update:
   - `MPESA_CONSUMER_KEY` - From Safaricom Daraja API
   - `MPESA_CONSUMER_SECRET` - From Safaricom Daraja API
   - `MPESA_INITIATOR_PASSWORD` - From Safaricom

### Step 3: Connect GitHub for Auto-Deployment
1. Go to Railway Dashboard
2. Select **hotelbook-api** service
3. Go to **Deployments** tab
4. Click **Connect Repository** → Select GitHub
5. Choose your `hotelbook` repository
6. Configure deployment settings:
   - Build Command: `npm run build:server`
   - Start Command: `npm run start`

### Step 4: Deploy Frontend
Option A - Separate Vercel Deployment (Recommended):
```bash
cd client
npm install -g vercel
vercel --prod
```
Then update `CLIENT_URL` in Railway with Vercel URL.

Option B - Deploy with Railway:
1. Create another Node.js service for frontend in Railway
2. Build and serve the static files

### Step 5: Final Verification
```bash
# From your local machine
npm run build       # Verify local build works
npm run test        # Verify tests pass

# Then push to trigger auto-deployment
git add .
git commit -m "Final configuration for Railway deployment"
git push origin main
```

## 🆘 Troubleshooting

**"Deployment failed"**
→ Check Railway logs: `npx railway logs --tail`

**"DATABASE_URL undefined"**
→ Verify postgres service is linked: `npx railway service postgres`

**"CORS error"**
→ Update CLIENT_URL to match your frontend deployment URL

**"OAuth credentials invalid"**
→ Replace placeholder values with real credentials from OAuth console
